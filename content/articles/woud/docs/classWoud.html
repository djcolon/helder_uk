Window

The window the simulation is displayed in and related items

If a need for a redraw is detected during the main loop needRedraw is set. At the start of the next iteration the window will be cleared. needRedraw will be set to false and doingRedraw will be set. needDraw will be set as well. This will cause every object being iterated over to be rendered to the screen. This means there can be a one-frame lag in drawing objects.

(USE_FASTADD) Spawns a tree within the bloomradius of another tree

(!USE_FASTADD) Spawns a tree within the bloomradius of another tree

Parameters
AParentTree to spawn within the bloomradius of

Generates x and y and does a check to see whether it falls outside the bloomradius. If so drops it and returns. Adds tree to the internal list but does not insert it into the scene Sets needDraw to true

Attention
This function will be used when USE_FASTADD is defined for the preprocessor
Parameters
AParentTree to spawn within the bloomradius of

Generate a random x and then calculates a maximum and minimum y for the spawned tree to be confined by a circle defined by the parent tree's position and the bloomradius. Adds tree to the internal list but does not insert it into the scene Sets needDraw to true

Attention
This function will be used when USE_FASTADD is not defined for the preprocessor

Tracks framerate when called every iteration.

Called every iteration. Increases the framerateBuffer by one. Gets the current time to see if a second has passed since last debuffering the framerate. If so it will write the buffer to framerate and reset the buffer to 0.

◆ run()

void Woud::run ( )

Main application loop.

Runs while gracefulShutdown has not been set Every iteration it will:

  • If it is not paused it will:
    • Update the draw status
    • Processes the framerate
    • Process lightnings
    • Process the trees
    • Rerender or draw if required
  • Get events from the window and handle them

void Woud::spreadFire ( Tree *  ATree)
private

Spreads fire from a burning tree.

Parameters
ATreeTree to spread fire from

Check whether the tree has missed too many chances to burn as defined in fireSpreadMissMax to save collision checks It then generates a rect around the tree and retrieves a list of all trees in that rect from the quadtree It will loop through the trees and first check if they ar enot already burning It will then check whether the tree is actually inside the burnradius. If so it sets the tree on fire and sets needDraw to true
